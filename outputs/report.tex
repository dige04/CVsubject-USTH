%--- ABSTRACT ---
\begin{abstract}
\noindent
We present a vision-first multi-modal system for hand gesture recognition and interactive game control, deployed entirely client-side in a standard web browser. Our approach begins with the cheapest visual signal---skeleton geometry extracted by MediaPipe---and progressively incorporates appearance features via a CNN, evaluating whether additional modalities improve recognition. Through a systematic comparison of methods of increasing complexity on the HaGRID v2 dataset (\num{23850}~samples, \num{13250}~subjects, 5~classes) under person-aware Group 5-Fold Cross-Validation, we find that a compact MLP (\num{14789}~parameters, \SI{227}{\kilo\byte}) operating on 60-dimensional skeleton landmarks achieves \SI{99.8}{\percent} accuracy, and that late fusion with a CNN yields no measurable improvement---a ceiling effect that constitutes an informative negative result about the relationship between task difficulty and fusion benefit. Beyond classification, we contribute a mathematical interaction model adapted from Huda et~al.~(2025) that uses discrete directional thresholds ($mov_x$, $mov_y$) and orthogonal tolerance zones ($tol$) to bridge frame-level predictions and robust game actions. The complete system runs at 30+~FPS on commodity hardware with no backend server.

\vspace{0.5em}
\noindent
\textbf{Keywords:} hand gesture recognition, multi-modal fusion, MediaPipe, mathematical interaction model, browser deployment, real-time control
\end{abstract}

% 1. INTRODUCTION
%=============================================================================
\section{Introduction}
\label{sec:intro}

Touchless interaction through hand gestures offers a natural modality for human--computer interaction (HCI), eliminating the need for specialized hardware beyond a commodity RGB webcam. Modern pose estimation frameworks such as MediaPipe~\citep{mediapipe2019,mediapipehands2020} have made real-time hand tracking practical even within web browsers, opening the door to gesture-controlled applications that require zero installation and zero backend infrastructure.

This paper takes a \textbf{vision-first} approach to hand gesture recognition. By ``vision-first,'' we mean a design philosophy that starts from the cheapest, fastest visual representation---the 2D/3D skeleton geometry that MediaPipe extracts in real time---and progressively adds richer but more expensive representations (RGB appearance features via a CNN) only if the simpler signal proves insufficient. This contrasts with approaches that begin with heavyweight end-to-end models and optimize downward. The vision-first principle produces a natural multi-modal architecture: the skeleton stream (MLP) and the appearance stream (CNN) are independent modalities derived from the same camera frame, combined via late fusion.

We address three research questions:
\begin{enumerate}
    \item[\textbf{Q1.}] \textbf{Method progression:} How do classification methods of increasing complexity---from hand-crafted heuristics through classical machine learning to deep neural networks---compare on the same gesture recognition task under person-aware evaluation?
    \item[\textbf{Q2.}] \textbf{Fusion benefit:} Does multi-modal fusion of skeleton and appearance features provide measurable improvement over the best unimodal method?
    \item[\textbf{Q3.}] \textbf{Classification-to-control:} How can frame-level gesture predictions be transformed into robust, jitter-free interactive control suitable for a real-time game?
\end{enumerate}

Our contributions are threefold:
\begin{enumerate}
    \item A \textbf{multi-modal comparison} of six methods on a 5-class subset of HaGRID v2~\citep{hagrid2022} with person-aware Group 5-Fold Cross-Validation, yielding an honest negative result: fusion provides no measurable benefit when both unimodal methods independently saturate accuracy---a boundary condition that is itself informative.
    \item A \textbf{mathematical interaction model} adapted from Huda et~al.~\citep{huda2025wheelchair} that uses discrete directional thresholds and a finite state machine with debounce buffers to translate noisy classifications into robust puzzle tile control.
    \item A \textbf{fully client-side browser deployment} using MediaPipe (WebAssembly) and ONNX Runtime Web, achieving 30+~FPS with a \SI{227}{\kilo\byte} model and zero server dependency.
\end{enumerate}

\paragraph{Related work.}
The HaGRID dataset~\citep{hagrid2022} provides the largest public benchmark for hand gesture recognition with 554K images across 18 classes; SOTA baselines on the full set include ResNet-152 (F1\,=\,95.5) with 60M parameters. Al~Farid et~al.~\citep{alfarid2022survey} survey vision-based gesture recognition systems, identifying real-time performance and deployment cost as persistent challenges. In applied gesture control, Nguyen et~al.~\citep{nguyen2025wheelchair} deploy YOLOv8n with MediaPipe preprocessing on an Nvidia Jetson Nano for wheelchair control (99.3\% accuracy, 6 gestures). Most relevantly, Huda et~al.~\citep{huda2025wheelchair} propose a mathematical model for wheelchair directional control using landmark distances, angular thresholds, and tolerance zones for filtering unintended movements, achieving 98.14\% accuracy at 21+~FPS with no deep learning. Our work extends their mathematical interaction paradigm from assistive wheelchair navigation to interactive game control.

\subsection{Dual-Representation Extraction}

From each image, we extract two parallel representations corresponding to our two modalities:

\paragraph{Skeleton landmarks (Modality~1).} MediaPipe HandLandmarker~\citep{mediapipehands2020} extracts 21 three-dimensional hand landmarks per detected hand. The raw 63-dimensional vector ($21 \times 3$) is normalized as follows:

\textbf{Translation normalization.} All coordinates are translated so the wrist (landmark~0) lies at the origin:
\begin{equation}
    l'_i = l_i - l_0, \quad \forall\, i \in \{0, 1, \ldots, 20\}
    \label{eq:translate}
\end{equation}

\textbf{Scale normalization.} Coordinates are divided by the Euclidean distance from the wrist to the middle-finger MCP joint (landmark~9):
\begin{equation}
    \hat{l}_i = \frac{l'_i}{\|l'_9\|_2}, \quad \forall\, i \in \{1, 2, \ldots, 20\}
    \label{eq:scale}
\end{equation}

\noindent The wrist landmark (now at the origin) is excluded, producing a \textbf{60-dimensional feature vector} per sample ($20 \times 3$). \Cref{fig:landmarks} shows the skeleton overlays for each gesture class, and \Cref{fig:tsne} visualizes the resulting feature space.

\paragraph{Hand crop images (Modality~2).} For training, we use \textbf{YOLO ground-truth bounding box annotations} provided by the HaGRID dataset to extract hand regions. For deployment, we use \textbf{MediaPipe detection bounding boxes} instead (since ground-truth annotations are unavailable at inference time). Cropped regions are resized to $224 \times 224$ RGB and normalized using ImageNet statistics ($\mu = [0.485, 0.456, 0.406]$, $\sigma = [0.229, 0.224, 0.225]$).

\subsection{Class Imbalance}

The dataset exhibits a 3:1 class imbalance: Frame and None contain \num{7950} samples each (due to multiple hands per image), while the remaining three classes have \num{2650} samples each. We address this by using \textbf{stratified group splits} within person-aware cross-validation, ensuring proportional class representation in each fold. We do not apply class weighting in the loss function: the 3:1 ratio is moderate, and all learned methods achieve per-class F1 above 99\% for every class including the minority classes, indicating that the imbalance does not degrade performance at this ratio. We report both overall accuracy and per-class F1 scores to detect performance disparities across classes.

\section{Multi-Modal Classification}
\label{sec:classification}

We evaluate six methods of increasing complexity under a unified evaluation protocol: \textbf{person-aware Group 5-Fold Cross-Validation} (scikit-learn's \texttt{GroupKFold}) with subject identifiers as group labels, guaranteeing that no individual appears in both training and test partitions within any fold.

\subsection{Skeleton Stream}
\label{sec:skeleton}

We evaluate three skeleton-based classifiers of increasing sophistication, all operating on the same 60-dimensional (or derived) landmark representation.

\paragraph{Heuristic classifier.} Approximately 20~manually calibrated threshold rules on geometric features: finger curl angles (proximal-intermediate-distal segment angles) and inter-landmark distances. Classification maps geometric conditions to gestures: all fingers extended $\to$ Open~Hand, all flexed $\to$ Fist, thumb-index proximity below threshold $\to$ Pinch. No training data is required.

\paragraph{Random Forest.} An ensemble of 100 decision trees~\citep{breiman2001rf} trained on a 24-dimensional engineered feature set comprising pairwise Euclidean distances between selected landmark pairs (fingertip-to-fingertip, fingertip-to-MCP, PIP joint distances). Implemented via \texttt{sklearn.ensemble.RandomForestClassifier} with maximum depth of~15.

\paragraph{MLP.} A feed-forward neural network operates directly on the full 60-dimensional normalized landmark vector:
\begin{equation}
    \text{Input}(60) \to \text{Dense}(128, \text{ReLU}) \to \text{Dropout}(0.3) \to \text{Dense}(64, \text{ReLU}) \to \text{Dropout}(0.3) \to \text{Dense}(5, \text{Softmax})
    \label{eq:mlp}
\end{equation}
\noindent The model has \num{14789}~trainable parameters (\SI{227}{\kilo\byte}). Training uses Adam~\citep{kingma2014adam} ($\text{lr} = 10^{-3}$), sparse categorical cross-entropy loss, and early stopping (patience~=~10).

\subsection{Appearance Stream}
\label{sec:appearance}

To exploit visual appearance information beyond skeleton geometry, we fine-tune MobileNetV3-Small~\citep{howard2019mobilenetv3} pretrained on ImageNet:
\begin{itemize}
    \item \textbf{Backbone:} MobileNetV3-Small feature extractor (first 8~blocks frozen)
    \item \textbf{Head:} AdaptiveAvgPool $\to$ Linear(576, 1024) $\to$ Hardswish $\to$ Dropout(0.2) $\to$ Linear(1024, 5)
    \item \textbf{Input:} $224 \times 224 \times 3$ RGB hand crops, ImageNet-normalized
    \item \textbf{Training:} Adam ($\text{lr} = 10^{-3}$), cross-entropy loss, 30~epochs per fold
    \item \textbf{Augmentation:} Random horizontal flip, rotation ($\pm15°$), color jitter
\end{itemize}

\noindent The CNN produces a 5-class softmax output representing the appearance modality. The model size is approximately \SI{4.3}{\mega\byte} (\SI{0.3}{\mega\byte} in ONNX format).

\subsection{End-to-End Detection Baseline}
\label{sec:yolo}

As an end-to-end baseline, we train YOLOv8n~\citep{jocher2023yolo} to simultaneously localize and classify hand gestures directly from full-resolution images ($640 \times 640$). The model has \num{3011823}~parameters (\SI{6.2}{\mega\byte}) and 8.2~GFLOPs. Training uses SGD ($\text{lr} = 0.01$, $\text{momentum} = 0.9$), cosine schedule ($\text{lrf} = 0.01$), 100~epochs with patience~20, batch size~16, and mixed precision on an NVIDIA A100-SXM4-80GB GPU. Unlike the classification methods above, YOLO performs \textbf{detection + classification} jointly, requiring detection metrics (mAP@50, mAP@50:95) rather than pure classification accuracy.

\subsection{Late Fusion}
\label{sec:fusion}

The fusion method combines the softmax outputs of the MLP (skeleton) and CNN (appearance) via a weighted average:
\begin{equation}
    P_{\text{fused}} = \alpha \cdot P_{\text{MLP}} + (1 - \alpha) \cdot P_{\text{CNN}}
    \label{eq:fusion}
\end{equation}
\noindent where $\alpha \in [0, 1]$ controls the relative contribution of each modality. The optimal $\alpha^*$ is determined via grid search on the training partition of each cross-validation fold. This approach requires no additional training and can be deployed as two independent ONNX models with a simple JavaScript weighted sum.

We also attempted a \textbf{learned fusion} approach, where the MLP and CNN feature vectors were concatenated and fed into a trainable classifier. This approach achieved only \SI{31.8}{\percent} ($\pm$\SI{5.2}{\percent}) accuracy---near random chance---due to the small size of the paired dataset (443~samples with both modalities available). This negative result demonstrates that learned fusion requires substantially more paired data than was available.

\subsection{Multi-Modal Finding: The Ceiling Effect}
\label{sec:ceiling}

The fusion result of \SI{100.0}{\percent} accuracy matches the CNN's unimodal performance exactly, providing \textbf{no measurable improvement}. This negative result is not a failure of the fusion method but a consequence of the \textbf{ceiling effect}: when both modalities independently achieve near-perfect accuracy on the same task, there is no residual error for fusion to correct.

Three factors explain why the five-class problem saturates:
\begin{enumerate}
    \item \textbf{Visual distinctiveness:} The five gestures---open hand, fist, pinch, frame, and no gesture---exhibit fundamentally different hand configurations that are easily separated in both skeletal and appearance feature spaces (as confirmed by the t-SNE visualization in \Cref{fig:tsne}).
    \item \textbf{Person-aware evaluation:} Group 5-Fold CV prevents identity leakage, confirming that saturation is genuine generalization rather than overfitting.
    \item \textbf{Clean data:} HaGRID images are captured under varied but controlled conditions with reliable annotations.
\end{enumerate}

This finding does not refute multi-modal fusion in general. On the full 18-class HaGRID taxonomy where SOTA is 95.5\% F1 (ResNet-152, 60M parameters), fusion would likely provide measurable benefit. Our result establishes a \textbf{boundary condition}: for $K$ visually distinct gesture classes where $K$ is small, a single modality suffices, and the research value lies in documenting \emph{where} this saturation occurs.

\paragraph{Statistical note.} With only $k = 5$ fold-level observations, a paired Wilcoxon signed-rank test between MLP (99.8\%) and CNN (100.0\%) accuracy does not reach statistical significance ($p > 0.05$). The 0.2 percentage-point gap is within the noise margin of fold-level variation, reinforcing the ceiling-effect interpretation: for this task, both modalities are functionally equivalent.

\section{Mathematical Model for Interactive Control}
\label{sec:mathmodel}

\subsection{The Classification-to-Control Gap}

Gesture classification assigns a discrete label $\hat{y}_t \in \{\text{open\_hand}, \text{fist}, \text{pinch}, \text{frame}, \text{none}\}$ to each video frame $t$. However, interactive control requires \textit{continuous, intentional} actions. Directly mapping raw per-frame predictions to game actions produces two failure modes:
\begin{enumerate}
    \item \textbf{Jitter:} Natural hand tremor causes the cursor position to fluctuate by 5--15 pixels per frame, triggering unintended actions in a free-drag mechanic.
    \item \textbf{Transient misclassification:} Even at 99.8\% accuracy, isolated frame-level errors cause erratic behavior---e.g., a momentary ``open\_hand'' prediction during a sustained pinch releases the tile prematurely.
\end{enumerate}

Huda et~al.~\citep{huda2025wheelchair} identified this same gap in wheelchair control and proposed a mathematical model using movement thresholds and tolerance zones. We adapt their paradigm from assistive wheelchair navigation to grid-based puzzle interaction.

\subsection{Gesture-to-Action Mapping}

\Cref{tab:gesture_actions} defines the mapping between the five gesture classes and game actions. This mapping was designed to feel intuitive: pinch grabs (analogous to physical pinching), open hand releases, fist triggers a timed action, and frame captures.

\textbf{Gesture} & \textbf{Game Action} & \textbf{Interaction Semantics} \\
\midrule

\subsection{Grid-Based Swipe Model}
\label{sec:swipe}

Adapted from the directional control model of Huda et~al.~\citep{huda2025wheelchair}, we implement a discrete swipe mechanic that replaces free-drag with threshold-gated tile swaps. Let $(x_{\text{init}}, y_{\text{init}})$ denote the cursor position when a grab begins, and $(x_{\text{cur}}, y_{\text{cur}})$ the current cursor position. We define the displacement:
\begin{equation}
    \Delta x = x_{\text{cur}} - x_{\text{init}}, \qquad \Delta y = y_{\text{cur}} - y_{\text{init}}
    \label{eq:displacement}
\end{equation}

Two threshold parameters govern the model:
\begin{itemize}
    \item $mov_x, mov_y$: \textbf{Movement thresholds} --- the minimum displacement in the intended direction required to trigger a swap. Set to $0.6 \times \text{tile\_size}$ in each axis.
    \item $tol$: \textbf{Orthogonal tolerance zone} --- the maximum displacement in the perpendicular direction permitted during a directional swipe. Set to $0.3 \times \min(\text{tile\_width}, \text{tile\_height})$.
\end{itemize}

\Cref{tab:swipe_conditions} defines the four directional swipe conditions, directly paralleling the logical condition table in Huda et~al.~\citep{huda2025wheelchair} (their Table~3).

\begin{table}[htbp]
\centering
\caption{Logical conditions for directional tile swapping. Each condition requires sufficient displacement in the intended direction \textit{and} bounded displacement in the orthogonal direction, following Huda et~al.'s formulation.}
\label{tab:swipe_conditions}
\begin{tabular}{@{}lccc@{}}
\toprule
\textbf{Direction} & \textbf{Primary Condition} & \textbf{Tolerance Condition} & \textbf{Tile Swap} \\
\midrule
Right   & $\Delta x > mov_x$   & $|\Delta y| < tol$ & $\text{tile}[i] \leftrightarrow \text{tile}[i+1]$ \\
Left    & $\Delta x < -mov_x$  & $|\Delta y| < tol$ & $\text{tile}[i] \leftrightarrow \text{tile}[i-1]$ \\
Down    & $\Delta y > mov_y$   & $|\Delta x| < tol$ & $\text{tile}[i] \leftrightarrow \text{tile}[i+C]$ \\
Up      & $\Delta y < -mov_y$  & $|\Delta x| < tol$ & $\text{tile}[i] \leftrightarrow \text{tile}[i-C]$ \\
\bottomrule
\end{tabular}
\end{table}

Upon a successful swap, the grab origin is updated to allow continuous multi-step swiping:
\begin{equation}
    (x_{\text{init}}, y_{\text{init}}) \leftarrow (x_{\text{cur}}, y_{\text{cur}})
    \label{eq:origin_update}
\end{equation}

The tolerance zone provides the critical HCI benefit: it filters out unintended off-axis hand jitter. A user intending to swipe right may exhibit small vertical tremor; the condition $|\Delta y| < tol$ ensures that only a clean horizontal displacement triggers the action. This directly mirrors the robustness that Huda et~al. achieve for wheelchair directional control, where unintended orthogonal movements could be dangerous.

\subsection{Finite State Machine}
\label{sec:fsm}

The interaction model is governed by a three-state finite state machine (FSM) with debounce-gated transitions. Let $g_t$ denote the raw gesture prediction at frame $t$, and $\bar{g}_t$ the \textbf{stable gesture}---defined as $g_t$ only if the last $N = 3$ consecutive predictions agree:
\begin{equation}
    \bar{g}_t =
    \begin{cases}
        g_t & \text{if } g_{t} = g_{t-1} = g_{t-2} \\
        \varnothing & \text{otherwise}
    \end{cases}
    \label{eq:debounce}
\end{equation}

\noindent This 3-frame debounce buffer suppresses transient misclassifications. \Cref{fig:fsm} illustrates the state machine transitions.

\begin{figure}[htbp]
\centering
\begin{tikzpicture}[
    state/.style={draw, rounded corners, fill=tolblue!15, minimum width=2.8cm, minimum height=1.2cm, align=center, font=\small\bfseries},
    transition/.style={-{Stealth[scale=0.8]}, thick, tolblue!70},
    lbl/.style={font=\scriptsize, fill=white, inner sep=2pt, align=center},
    node distance=3.5cm,
]
    \node[state] (idle) {IDLE};
    \node[state, right=4.5cm of idle] (grab) {GRABBING};
    \node[state, below=3cm of idle] (fist) {FIST HOLD};

    \draw[transition] (idle.east) -- node[lbl, above] {pinch + cursor over tile} (grab.west);
    \draw[transition] (grab.south west) -- node[lbl, right, pos=0.4] {open hand or none} (idle.north east);
    \draw[transition] (idle.south) -- node[lbl, right] {fist} (fist.north);
    \draw[transition] (fist.east) -- ++(1.5,0) |- node[lbl, right, pos=0.25] {not fist or timer expires} (idle.east);

\end{tikzpicture}
\caption{Finite state machine for gesture-based interaction. All transitions require 3-frame debounce stability ($\bar{g}_t$). The GRABBING state continuously evaluates swipe conditions (\Cref{tab:swipe_conditions}) against the grid-based threshold model.}
\label{fig:fsm}
\end{figure}

\paragraph{State transitions in detail.}

\begin{enumerate}
    \item \textbf{IDLE $\to$ GRABBING:} When $\bar{g}_t = \text{pinch}$ and the smoothed cursor is over a puzzle tile, the system records the grab origin $(x_{\text{init}}, y_{\text{init}})$ and locks the tile. The user's hand position is now tracked for directional swipe evaluation.

    \item \textbf{GRABBING (continuous):} While in this state, each frame evaluates the four swipe conditions in \Cref{tab:swipe_conditions}. If a condition is met, the tile is swapped with its neighbor, and the origin is reset per \Cref{eq:origin_update}. This allows chained multi-step swiping without releasing the pinch gesture.

    \item \textbf{GRABBING $\to$ IDLE:} When $\bar{g}_t \in \{\text{open\_hand}, \text{none}\}$, the tile lock is released. The 3-frame requirement prevents accidental release from a single-frame misclassification.

    \item \textbf{IDLE $\to$ FIST\_HOLD:} When $\bar{g}_t = \text{fist}$, a dwell timer begins. If the fist is held for 1.5~seconds, the puzzle resets. This dwell activation prevents accidental resets from transient fist detections.

    \item \textbf{FIST\_HOLD $\to$ IDLE:} If the gesture changes before the timer expires, the reset is cancelled.
\end{enumerate}

\subsection{Smoothed Cursor with Adaptive Alpha}
\label{sec:cursor}

The raw cursor position is computed as the midpoint of the thumb tip (landmark~4) and index fingertip (landmark~8), providing a natural ``pinch point'' for interaction:
\begin{equation}
    \mathbf{p}_{\text{raw}}^{(t)} = \frac{\mathbf{l}_4^{(t)} + \mathbf{l}_8^{(t)}}{2}
    \label{eq:rawcursor}
\end{equation}

To reduce jitter without introducing latency, we apply an exponential moving average with an \textbf{adaptive smoothing factor}:
\begin{equation}
    \alpha_t =
    \begin{cases}
        1.0 & \text{if } \|\mathbf{p}_{\text{raw}}^{(t)} - \mathbf{p}_{\text{smooth}}^{(t-1)}\| > 100\text{px} \\
        0.4 & \text{otherwise}
    \end{cases}
    \label{eq:adaptive_alpha}
\end{equation}
\begin{equation}
    \mathbf{p}_{\text{smooth}}^{(t)} = (1 - \alpha_t) \cdot \mathbf{p}_{\text{smooth}}^{(t-1)} + \alpha_t \cdot \mathbf{p}_{\text{raw}}^{(t)}
    \label{eq:ema}
\end{equation}

The adaptive threshold of 100 pixels distinguishes between two regimes: (1)~small displacements from hand tremor, which are heavily smoothed ($\alpha = 0.4$), and (2)~large intentional movements (e.g., moving to a distant tile), which pass through with no smoothing ($\alpha = 1.0$) to avoid perceptible lag.

\subsection{Browser-Based Deployment}

The deployed application operates entirely client-side:
\begin{itemize}
    \item \textbf{Hand tracking:} MediaPipe tasks-vision (WebAssembly) provides real-time landmark extraction.
    \item \textbf{Inference:} Both MLP and CNN models are exported to ONNX format and run via ONNX Runtime Web. The MLP uses the WASM backend; the CNN attempts WebGPU first, falling back to WASM.
    \item \textbf{Fusion:} A simple JavaScript weighted sum combines softmax outputs ($\alpha = 0.7$, MLP-dominant).
    \item \textbf{Interaction model:} The state machine, swipe thresholds, debounce buffer, and adaptive cursor are implemented in approximately 150~lines of JavaScript.
    \item \textbf{Application:} An HTML5 Canvas-based jigsaw puzzle translates gesture predictions into interactive game actions.
\end{itemize}

\subsection{Deployment-Specific Adaptations}
\label{sec:deployment_note}

The deployed system differs from the benchmarking pipeline (\Cref{fig:pipeline}) in two ways:

\begin{enumerate}
    \item \textbf{MLP-only classification.} Based on the ceiling-effect finding (\Cref{sec:ceiling}), the deployed system uses the MLP (skeleton) path as the primary classifier. The CNN path is loaded as an optional fallback but is not required for reliable recognition.
    \item \textbf{Two-hand frame detection.} The ``frame'' gesture requires two hands forming a rectangle---a configuration that is reliably detectable by checking whether MediaPipe reports two simultaneous hand detections. In the deployed system, this geometric heuristic bypasses the MLP classifier for the frame class, assigning ``frame'' with high confidence whenever two hands are present. The MLP is trained and evaluated on all five classes (including frame from single-hand crops) in the benchmarking study, but the deployment heuristic provides more robust frame detection in practice because it leverages the defining structural property of the gesture. This creates a minor inconsistency: the deployed classifier is effectively a 4-class MLP augmented by a 2-hand heuristic, while the benchmarking study evaluates a 5-class MLP.
\end{enumerate}

\section{Discussion and Conclusion}
\label{sec:conclusion}

\subsection{Answering the Research Questions}

\textbf{Q1 --- Method progression.} Methods of increasing complexity yield monotonically increasing accuracy: from 0.99\% (heuristic, which fails due to dimensionality mismatch between rules designed for 3D coordinates and 2D-projected data) through 94.1\% (Random Forest on 24 engineered distances) to 99.8\% (MLP on full 60D landmarks) and 100.0\% (CNN on RGB crops). This confirms that learned representations consistently outperform hand-crafted features on this dataset, and that the full landmark vector retains discriminative information lost by feature engineering.

\textbf{Q2 --- Fusion benefit.} Late fusion of skeleton (MLP) and appearance (CNN) modalities achieves \SI{100.0}{\percent} accuracy, matching the CNN's unimodal result exactly. This negative result establishes a boundary condition: for five visually distinct gesture classes, each modality independently saturates performance, leaving no residual error for fusion to correct. The finding is informative precisely because it quantifies \emph{where} multi-modal fusion ceases to add value---when task difficulty is below the threshold that any single modality can solve. Fusion benefits would be expected on harder tasks (e.g., the full 18-class HaGRID taxonomy where SOTA is 95.5\% F1).

\textbf{Q3 --- Classification to control.} The mathematical interaction model adapted from Huda et~al.~\citep{huda2025wheelchair}---using movement thresholds ($mov_{x,y} = 0.6 \times \text{tile\_size}$), orthogonal tolerance zones ($tol = 0.3 \times \text{tile\_size}$), a 3-state FSM with 3-frame debounce, and an adaptive EMA cursor---successfully bridges the gap between frame-level classification and intentional game actions. The tolerance zone filters hand jitter, the debounce buffer suppresses transient misclassifications, and the adaptive cursor balances responsiveness against stability.

\subsection{Summary of Contributions}

\Cref{tab:bestmethod} summarizes the best-performing method for each evaluation criterion.

The vision-first approach validates a practical design principle: start with the cheapest signal (skeleton geometry), verify its sufficiency, and add richer modalities only when the task demands it. For our 5-class problem, the \SI{227}{\kilo\byte} MLP provides the optimal trade-off between accuracy, model size, and inference speed for edge deployment.

\subsection{Limitations}

\begin{enumerate}
    \item \textbf{Gesture vocabulary:} The five target classes are visually distinct and represent a low-difficulty classification problem. A finer-grained gesture set (e.g., ASL signs, subtle finger configurations) would stress-test both the classifier and the interaction model, and would likely reveal meaningful fusion benefits.
    \item \textbf{No temporal modeling:} Our system treats each frame independently (aside from debounce). Recurrent or attention-based models over landmark sequences could improve recognition of dynamic gestures (e.g., waving, circular motions).
    \item \textbf{Fixed interaction thresholds:} The values $mov_x$, $mov_y$, and $tol$ are fixed proportions of tile size. Adaptive calibration per user, as Huda et~al. do with their threshold determination from hand samples, could improve flexibility for users with varying dexterity.
    \item \textbf{Dataset conditions:} HaGRID images are captured under varied but relatively controlled conditions. Performance in challenging environments (extreme lighting, motion blur, heavy occlusion) remains to be validated.
\end{enumerate}

\subsection{Future Directions}

Future work includes: (1)~extending the gesture vocabulary to the full 18-class HaGRID taxonomy where multi-modal fusion is expected to provide measurable benefit; (2)~incorporating temporal sequence modeling (LSTM or Transformer over landmark sequences) for dynamic gesture recognition; (3)~adaptive threshold calibration from per-user hand samples following Huda et~al.'s methodology; and (4)~a formal user study evaluating the mathematical interaction model's usability compared to free-drag alternatives.
